### Data Processing
- **Observation/Action:** Stratified K-Fold CV uses a composite label `combined_y = y[:, 0] * 2 + y[:, 1]` to preserve joint target distributions. Final probabilities are clipped to `[0.01, 0.99]`. Explicitly avoids `QuantileTransformer`/`StandardScaler` for tree ensembles.
- **ML Rationale (Why it works):** Multi-label targets are rarely independent; their joint distribution (00, 01, 10, 11) often contains rare but highly informative co-occurrence patterns. Composite encoding ensures each fold maintains proportional representation of these joint states, preventing validation leakage from distributional shifts. Clipping stabilizes downstream meta-learning by bounding extreme probability tails that trees can produce due to synthetic artifacts. Avoiding scaling is mathematically sound for GBDTs: they are invariant to monotonic transformations, and scaling can inadvertently amplify generation noise or distort optimal split thresholds in synthetic feature spaces.
- **Guiding Principle:** When targets are correlated, stratify on their joint combinatorial state, not individual margins. Preserve raw feature magnitudes for tree-based models unless dealing with distance/similarity metrics. Always bound meta-learner inputs to prevent numerical instability and ensure smooth probability blending.

### Feature Engineering
- **Observation/Action:** Computes row-wise aggregates across all input features: `nan_count`, `mean`, `std`, `min`, `max`, `skewness`, and `kurtosis`. Handles missing values natively via `np.nan*` functions.
- **ML Rationale (Why it works):** In synthetically generated tabular data, individual features often contain weak or noisy signals due to the generative process (e.g., decoder bottlenecks, noise injection). However, the *joint row profile* frequently encodes strong latent structure. Row-wise statistics act as a low-dimensional manifold projection, capturing an instance's "global state". Skew and kurtosis detect distributional anomalies per sample, which often correlate with target certainty. Explicitly counting NaNs treats missingness as an information-bearing signal, which is highly predictive in synthetic datasets where masking or generation failures are non-random.
- **Guiding Principle:** For high-dimensional or weakly-correlated tabular features, engineer instance-level (row-wise) statistical summaries. These features are highly robust to per-column noise and capture latent sample states. Treat missingness patterns as explicit features; in synthetic or pipeline-generated data, the *structure* of missing data often carries more signal than the imputed values.

### Model Architecture & Training
- **Observation/Action:** Trains three distinct GBDT frameworks (LightGBM, CatBoost, XGBoost) with identical, heavily regularized hyperparameters (`max_depth=4`, high L1/L2 penalties, `min_child_weight=80`, `subsample=0.8`). Stacks predictions using a shallow `LGBMRegressor` meta-learner.
- **ML Rationale (Why it works):** Each GBDT framework has fundamentally different inductive biases: LGBM uses histogram-based leaf-wise growth, XGBoost uses exact/approximate level-wise splits with second-order gradients, and CatBoost employs ordered boosting to mitigate target leakage. Their error surfaces are decorrelated. Heavy regularization and shallow trees force models to learn low-bias, robust patterns rather than memorizing high-variance synthetic artifacts (a common failure mode in DL-generated tabular data). Using a regressor as the meta-learner treats base probabilities as continuous blending weights, allowing smooth, non-linear interpolation without hard classification thresholds.
- **Guiding Principle:** Maximize ensemble diversity by combining algorithmically distinct frameworks, not just hyperparameter variants. When training on synthetic or noisy data, prioritize aggressive regularization and shallow trees to suppress overfitting to generation artifacts. Use a continuous meta-learner for stacking to learn optimal, context-dependent weighting of base model confidence.

### Validation & Evaluation
- **Observation/Action:** Implements strict 5-fold Out-of-Fold (OOF) prediction generation for stacking. Monitors Train vs. Validation AUC gap as an automated overfitting diagnostic. Uses early stopping with a generous patience (150 rounds).
- **ML Rationale (Why it works):** OOF predictions guarantee zero data leakage between base and meta models, which is critical for unbiased stacking. Synthetic datasets often contain spurious, dataset-specific correlations that don't generalize; a widening Train-Val AUC gap is a reliable heuristic that the model is fitting generation noise rather than true signal. Generous early stopping patience allows models to navigate noisy validation landscapes without premature convergence, while still capping overfitting.
- **Guiding Principle:** Always generate meta-learner inputs via strict OOF predictions to prevent target leakage. Implement automated Train-Val metric gap monitoring as a real-time diagnostic for synthetic data generalization. Calibrate early stopping patience to dataset size and expected noise levels; overly tight patience causes underfitting on noisy folds, while excessive patience invites overfitting.
