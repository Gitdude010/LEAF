### Data Processing
- **Observation/Action:** 16-bit grayscale MRI scans are linearly normalized by dividing by 65535, resized to a fixed 384×384 resolution, and the single channel is stacked into 3 channels. Original dimensions are preserved to reverse-transform predictions before RLE encoding.
- **ML Rationale (Why it works):** Medical MRI data contains high dynamic range where subtle intensity gradients define tissue boundaries. Linear scaling preserves the full signal-to-noise ratio without clipping diagnostically relevant low-intensity regions (unlike min-max or histogram equalization). Channel stacking bridges the modality gap between single-channel medical data and RGB-pretrained ImageNet backbones, enabling transfer learning. Tracking native dimensions is mathematically necessary because the Hausdorff distance metric is scale-sensitive; computing it on downsampled masks would artificially penalize boundary errors.
- **Guiding Principle:** Normalize high-bit-depth medical data linearly to preserve diagnostic contrast; adapt input channels to unlock pretrained representations; always decouple computational resolution from evaluation resolution to maintain metric fidelity and avoid scale-induced metric bias.

### Feature Engineering
- **Observation/Action:** No hand-crafted features are extracted. Spatial generalization is enforced via randomized horizontal/vertical flips and 90° rotations. Data splits are strictly performed at the patient-case level, ignoring individual slice indices.
- **ML Rationale (Why it works):** In dense pixel-wise tasks, convolutional networks inherently learn hierarchical spatial filters, making manual feature extraction redundant and often detrimental. Geometric augmentations simulate anatomical variability and scanner positioning differences, forcing the model to learn orientation-invariant tissue patterns rather than memorizing coordinate priors. Case-level splitting is critical because adjacent slices and multi-day scans from the same patient exhibit severe spatial and temporal autocorrelation; random slice splits leak patient-specific anatomy into validation, producing wildly optimistic performance estimates.
- **Guiding Principle:** Replace manual feature engineering with spatially diverse augmentations that mirror real-world acquisition variability. Always enforce patient/study-level partitioning in cross-validation to measure true clinical generalization and prevent spatial data leakage.

### Model Architecture
- **Observation/Action:** Implements a UNet-style encoder-decoder using a `timm` ResNet34 backbone, with explicit skip connections, BatchNorm, and ReLU activations. Gradient checkpointing is explicitly disabled to avoid autograd conflicts with inplace operations.
- **ML Rationale (Why it works):** The UNet topology is mathematically optimal for segmentation: the encoder captures global contextual semantics while the decoder restores spatial precision via skip connections, which mitigate the information loss inherent in downsampling. ResNet34 provides a favorable capacity-to-compute ratio, preventing overfitting on limited medical datasets while retaining sufficient receptive field. Disabling gradient checkpointing avoids inplace-activation autograd errors that can destabilize backward passes in certain framework versions, ensuring stable gradient flow through the decoder.
- **Guiding Principle:** Prioritize encoder-decoder architectures with skip connections for dense prediction tasks. Match backbone depth to dataset size and compute budget rather than maximizing theoretical capacity. Validate framework compatibility for memory-saving techniques to ensure training stability and gradient integrity.

### Validation & Training Strategy
- **Observation/Action:** Uses Focal Tversky Loss with asymmetric weighting (α=0.7, β=0.3, γ=0.75), gradient accumulation (4 steps), mixed-precision training, and explicit per-batch Dice calculation. Predictions are thresholded at 0.5, and validation strictly uses case-level splits.
- **ML Rationale (Why it works):** Medical segmentation suffers from extreme foreground-background imbalance. Focal Tversky asymmetrically penalizes false negatives (clinically critical for missing organs) while the focal term down-weights easy background examples, focusing optimization on hard boundary regions. Gradient accumulation simulates larger effective batch sizes, stabilizing BatchNorm statistics and reducing gradient variance without exceeding VRAM. Explicit Dice tracking aligns the optimization trajectory with the competition metric, preventing loss-metric divergence. Note: Since Hausdorff distance (60% weight) is not directly optimized in the loss, the model relies on Dice-driven boundary precision, which is a known trade-off in this setup.
- **Guiding Principle:** Design loss functions that explicitly address class imbalance and boundary sensitivity. Use gradient accumulation to decouple hardware limits from optimization stability. Always monitor the exact evaluation metric during training to detect loss-metric misalignment early, and consider boundary-aware losses or morphological post-processing when distance-based metrics dominate the leaderboard.
