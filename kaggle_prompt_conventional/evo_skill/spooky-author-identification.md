### Data Processing
- **Observation/Action:** Dynamic sequence padding via a custom `collate_fn` paired with `StratifiedKFold` splitting on author labels.
- **ML Rationale (Why it works):** Literary excerpts vary significantly in length. Fixed-length padding introduces excessive `[PAD]` tokens, which dilute attention weights and degrade gradient signal-to-noise ratios. Dynamic padding minimizes padding overhead, allowing the model to focus compute on actual tokens. Stratification guarantees each fold preserves the exact class distribution, preventing skewed validation metrics that could mislead early stopping or hyperparameter selection.
- **Guiding Principle:** Always implement dynamic batching for sequence data to optimize VRAM and attention efficiency. Pair with stratified sampling when class balance or stylistic distribution impacts evaluation stability, ensuring validation folds are statistically representative of the target population.

### Feature Engineering
- **Observation/Action:** Dual-pooling strategy that concatenates mean-pooled and max-pooled representations from the transformer's final hidden state before the classification head.
- **ML Rationale (Why it works):** Authorship attribution relies on two orthogonal signals: pervasive stylistic tone (sentence rhythm, average syntactic complexity) and salient lexical markers (rare vocabulary, archaic phrasing, distinctive punctuation). Mean pooling aggregates global contextual semantics, while max pooling isolates highly activated, stylistically distinctive tokens. Concatenating both provides the classifier with complementary latent features that a single pooling method (or `[CLS]` token alone) would miss.
- **Guiding Principle:** For tasks sensitive to both holistic context and localized discriminative signals (e.g., authorship, sentiment, forensic linguistics), combine multiple pooling strategies (mean, max, CLS, or attention-weighted) to construct a richer, multi-scale representation before classification.

### Model
- **Observation/Action:** Fine-tuning `microsoft/deberta-v3-base` with Layer-wise Learning Rate Decay (LLRD), gradient accumulation, label smoothing (`0.1`), and gradient clipping (`1.0`).
- **ML Rationale (Why it works):** DeBERTa-v3's disentangled attention and relative positional encoding excel at capturing long-range stylistic dependencies and archaic syntactic structures common in 19th-century horror. LLRD applies progressively smaller learning rates to lower transformer layers, preserving foundational linguistic priors while allowing higher layers to rapidly adapt to author-specific patterns. Gradient accumulation simulates larger effective batch sizes for stable optimization under VRAM constraints. Label smoothing penalizes overconfident predictions, directly aligning optimization with the log-loss metric by keeping probability distributions well-calibrated and robust to noisy or ambiguous excerpts.
- **Guiding Principle:** When adapting large pre-trained models to domain-specific or stylistic tasks, use LLRD to prevent catastrophic forgetting of lower-layer syntax. Pair with label smoothing and gradient clipping when optimizing for probabilistic metrics (log loss, Brier score) to improve calibration, reduce overconfidence penalties, and stabilize training dynamics.

### Validation
- **Observation/Action:** Out-of-Fold (OOF) prediction tracking, explicit log-loss clipping/row-normalization matching the competition metric, and fold-averaged test-time predictions.
- **ML Rationale (Why it works):** OOF predictions guarantee that every training sample is evaluated by a model that never saw it during training, providing an unbiased generalization estimate. Explicitly clipping probabilities to `[1e-15, 1-1e-15]` and row-normalizing during validation exactly replicates the evaluation pipeline, preventing metric drift and ensuring validation scores correlate 1:1 with leaderboard results. Averaging predictions across CV folds reduces model variance, smooths out fold-specific biases, and leverages ensemble diversity for better calibration.
- **Guiding Principle:** Always mirror the exact evaluation metric's preprocessing (clipping, scaling, rescaling) inside the validation loop to prevent optimization-validation mismatch. Use OOF predictions for reliable model selection, and ensemble multiple CV folds at inference to maximize robustness, reduce variance, and improve probabilistic calibration.
